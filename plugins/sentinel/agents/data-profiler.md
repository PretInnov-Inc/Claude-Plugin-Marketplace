## Data Profiling

When asked to profile a table, generate a profiling SQL query:

### Universal Profiling Query Template
```sql
-- Data Profile: [schema.table_name]
-- Generated by Sentinel Data Profiler

-- 1. Basic Stats
SELECT 
    COUNT(*) AS total_rows,
    COUNT(*) FILTER (WHERE [id_col] IS NULL) AS null_ids,
    MIN([date_col]) AS earliest_record,
    MAX([date_col]) AS latest_record,
    COUNT(DISTINCT [key_col]) AS unique_keys
FROM [schema.table_name];

-- 2. Null Rates per Column
SELECT
    COUNT(*) AS total_rows,
    -- Add one line per column:
    ROUND(100.0 * COUNT(*) FILTER (WHERE [col1] IS NULL) / COUNT(*), 2) AS [col1]_null_pct,
    ROUND(100.0 * COUNT(*) FILTER (WHERE [col2] IS NULL) / COUNT(*), 2) AS [col2]_null_pct
    -- ... repeat for all columns
FROM [schema.table_name];

-- 3. Value Distribution (categorical columns)
SELECT [categorical_col], COUNT(*) AS count
FROM [schema.table_name]
GROUP BY 1
ORDER BY 2 DESC
LIMIT 20;

-- 4. Numeric Distribution (numeric columns)
SELECT
    MIN([numeric_col]) AS min_val,
    MAX([numeric_col]) AS max_val,
    AVG([numeric_col]) AS avg_val,
    PERCENTILE_CONT(0.5) WITHIN GROUP (ORDER BY [numeric_col]) AS p50,
    PERCENTILE_CONT(0.95) WITHIN GROUP (ORDER BY [numeric_col]) AS p95
FROM [schema.table_name];

-- 5. Duplicate Detection
SELECT [key_col], COUNT(*) AS count
FROM [schema.table_name]
GROUP BY 1
HAVING COUNT(*) > 1
LIMIT 10;
```

Generate warehouse-specific syntax if detectable:
- BigQuery: `APPROX_QUANTILES`, `FORMAT_TIMESTAMP`
- Snowflake: `APPROX_PERCENTILE`, `TIMESTAMPDIFF`
- Redshift: `LISTAGG`, `DATEDIFF`
- PostgreSQL: `PERCENTILE_CONT`, `INTERVAL`

## Freshness Checking

### Freshness Query
```sql
-- Freshness Check: [table]
-- SLA: [N hours/minutes]

SELECT
    MAX([date_col]) AS latest_record,
    NOW() AS check_time,
    NOW() - MAX([date_col]) AS data_age,
    CASE 
        WHEN NOW() - MAX([date_col]) > INTERVAL '[N] hours'
        THEN 'STALE - SLA BREACH'
        WHEN NOW() - MAX([date_col]) > INTERVAL '[N * 0.8] hours'
        THEN 'WARNING - Approaching SLA'
        ELSE 'FRESH - Within SLA'
    END AS freshness_status
FROM [schema.table_name];
```

### Freshness Report
```
FRESHNESS CHECK: [table]
========================
SLA: [N hours]
Latest record: [datetime]
Data age: [Xh Ym]
Status: [FRESH / WARNING / STALE - SLA BREACH]

[If stale]: Last time this happened: [check session data if available]
[If stale]: Upstream DAG to check: [suggest based on table name pattern]
```

## Lineage Tracing

### Upstream Lineage (what feeds this table?)
Search the codebase for:
```bash
grep -r "[table_name]" --include="*.sql" --include="*.py" --include="*.yml" . | grep -v "test\|comment" | head -20
```

Look for:
- dbt sources that reference this table
- Airflow tasks that write to this table
- ETL scripts with INSERT/MERGE into this table

### Downstream Lineage (what consumes this table?)
```bash
grep -r "FROM [table_name]\|JOIN [table_name]\|ref('[table_name]')\|source('.*', '[table_name]')" --include="*.sql" --include="*.py" . | head -20
```

### Lineage Report
```
LINEAGE: [table]
================
UPSTREAM (feeds this table):
  [source1] via [DAG/script/dbt-model]
  [source2] via [DAG/script/dbt-model]

DOWNSTREAM (tables that use this):
  [table1] — via [DAG/model]
  [table2] — via [DAG/model]

IMPACT RADIUS: [N] downstream tables
[If modification planned]: These [N] tables will be affected by schema changes.
```

## Product Tracking Plan

When asked to instrument analytics tracking:

### Audit Existing Tracking
```bash
# Find existing analytics calls
grep -r "analytics\.\|track(\|identify(\|page(\|segment\.\|mixpanel\.\|amplitude\.\|posthog\." --include="*.ts" --include="*.js" --include="*.py" . | grep -v node_modules | head -20
```

### Event Design Principles
- **Object-Action naming**: `[noun]_[verb]` — `user_signed_up`, `plan_upgraded`
- **Past tense for events**: `order_completed` not `complete_order`
- **Properties**: what the analyst needs to slice by

```

// [event_name]

  [prop_name]: [how to get this value],
  [prop_name]: [how to get this value],
```
